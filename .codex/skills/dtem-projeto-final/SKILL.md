# Skill: Projeto Final Hands-on DTEM

Use esta skill quando o usuário pedir:
- projeto final;
- desafio final;
- projeto para portfólio;
- simulação corporativa;
- ambiente real de trabalho;
- sprint completa;
- sistema fictício;
- case completo.

# Objetivo

Criar um projeto final altamente prático que simule o funcionamento real de uma empresa de tecnologia.

O projeto deve permitir que os alunos atuem como:
- QA;
- Product Owner;
- Desenvolvedor;
- Time Ágil.

# Público-alvo

Iniciantes em tecnologia e pessoas em transição de carreira.

# Estrutura obrigatória

O projeto SEMPRE deve conter:

1. Nome do projeto
2. Contexto da empresa fictícia
3. Problema de negócio
4. Objetivo do sistema
5. Escopo funcional
6. Perfis envolvidos
7. Fluxo do sistema
8. Backlog
9. User Stories
10. Critérios de aceite
11. Casos de teste
12. Bugs simulados
13. Organização da Sprint
14. Daily simulada
15. Planejamento ágil
16. Fluxo SDLC
17. Simulação de deploy
18. Entrega final
19. Apresentação estilo empresa
20. Checklist de avaliação

# O projeto deve ensinar

- Como funciona uma empresa tech;
- Como times interagem;
- Como funciona uma Sprint;
- Como bugs são tratados;
- Como QA trabalha;
- Como Product Owner trabalha;
- Como funciona backlog;
- Como acontece um deploy;
- Como áreas se conectam.

# Deve incluir

- Diagramas Mermaid;
- Fluxogramas;
- Cenários reais;
- Situações corporativas;
- Simulações de problemas reais;
- Priorização de backlog;
- Comunicação entre áreas;
- Gestão de bugs;
- Casos reais inspirados no mercado.

# Exemplos de projetos

- Sistema de agendamento;
- Sistema de delivery;
- Banco digital;
- Sistema de chamados;
- Plataforma EAD;
- E-commerce;
- App financeiro;
- Plataforma de RH.

# Formato de saída

Sempre gerar:
- Markdown profissional;
- Estrutura organizada;
- Material rico;
- Arquivos reutilizáveis para portfólio.

# Regra principal

O aluno deve terminar o projeto com artefatos reais para:
- LinkedIn;
- GitHub;
- currículo;
- entrevistas;
- portfólio profissional.
